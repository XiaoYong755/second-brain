# Douyin Script

## Topic

我是怎么用 Karpathy 的 LLM wiki 思路，搭了一套真正能长期使用的本地第二大脑。

## Positioning

目标不是讲“我搭了一个多炫的系统”，而是让粉丝听完以后明白：

- 为什么很多人的知识库越记越乱
- 一个好用的第二大脑到底该怎么想
- 今天这套结构为什么值得抄

## Short Version Script

### Opening

如果你也搭过知识库，你大概率遇到过一个问题：
记的时候很兴奋，过几天再打开，根本不知道从哪继续。

我今天花了一整天，参考 Karpathy 的 LLM wiki 思路，搭了一套纯本地的第二大脑。搭完以后我最大的感受不是“终于有系统了”，而是我第一次觉得，这东西真的能长期用下去。

### The Problem

很多人的知识库不好用，不是因为工具不行，而是因为一开始就搞错了重点。

大家常见的做法是：

- 一上来先分很多目录
- 收藏很多文章和链接
- 记很多碎片笔记

最后的结果就是，东西越来越多，但真正能复用的理解越来越少。

### The Core Idea

我这次最核心的一个判断是：

第二大脑不能只是资料仓库，它必须有一个“理解层”。

所以我最后没有做纯 PARA，也没有只做 LLM 知识库，而是做了一套混合结构：

- `Inbox` 收碎片
- `Library` 放资料
- `Wiki` 放长期理解
- `Projects` 接现实推进
- `Journal` 接时间上下文

这套结构最关键的是：

`Library` 记录你看到了什么，`Wiki` 记录你现在怎么理解它。

### Why Karpathy Was Useful

Karpathy 那个 LLM wiki 给我最大的启发，不是“列很多知识点”，而是：

要把知识沉淀成长期维护的页面，而不是只做一次性的检索和摘抄。

这意味着你以后不是不断往里塞材料，而是不断把重要内容重写、压缩、链接起来。

### What I Built Today

我今天具体搭了 5 层东西：

- 顶层目录结构
- 首页和导航页
- 常用模板
- Git 和 GitHub 版本管理
- 第一批 LLM 核心页面

我连 `Transformer`、`Attention`、`Tokenization`、`Pretraining`、`Post-Training`、`Reasoning` 这几页都先补上了。

所以现在它已经不是一个空壳，而是已经有骨架了。

### Where I Got Stuck

当然中间也不是完全顺的。

最卡的地方其实不是知识结构，而是基础设施：

- 本地一开始不是 Git 仓库
- GitHub 没有现成认证
- 后来还补了 SSH key 和远程同步

这也让我意识到一件事：

真正能长期用的第二大脑，不只是目录清楚，还要能版本管理、能回溯、能同步。

### Biggest Takeaway

我今天最大的收获是：

一个第二大脑要想真正长期使用，不是越复杂越好，而是要满足 3 个条件：

- 你每次打开都知道从哪开始
- 你每条新信息都知道放哪里
- 你能持续把“看到的东西”提炼成“自己的理解”

### Closing

如果你也想搭自己的第二大脑，我建议你不要先纠结工具，也不要先抄一堆目录。

先问自己 3 个问题：

- 你的知识最终要沉淀成什么
- 新信息应该先流到哪里
- 你怎么把资料变成长期理解

把这 3 个问题想清楚，工具反而变成最不重要的部分。

如果你想，我下一条可以继续把我这套结构、模板和首页导航拆开讲细一点。
